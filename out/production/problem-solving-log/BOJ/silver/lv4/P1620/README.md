## 🤨 접근
### 🧬 로직
- 포켓몬을 map에 저장한다.
    - 이때 map은 번호-이름 맵과 이름-번호 맵 두 개를 만든다
- 입력값을 받아 각각의 맵의 키인지 확인하고, 키인 맵에 대해 값을 찾아 출력한다.

### 📑 새로운 문법(API)
<aside>

`Character.isDigit()` : 문자가 숫자인지 확인

</aside>


<br>

## 🤯 실수
### 시간 초과
- 처음엔 map을 하나만 만들고 값으로 key를 찾는 함수를 따로 만들었다. 그러나 그렇게 하니 시간초과가 발생해서, 그냥 map을 두 개 만들었다.

  <br>

## ✨ 소회
- 간단한 로직이어도 효율적 접근을 위해 고려해야할 요소가 있다.
- 처음 접근한 value로 key 찾기 방식은 모든 엔트리를 확인해야하므로 O(N)의 시간복잡도가 걸린다. 하지만 메모리를 내어 주고 배열이나 맵을 하나 더 만들면 O(1)의 시간복잡도를 유지할 수 있다.
