## 🤨 접근
### 🧬 로직
- 주어진 schedules를 기반으로 출근 시간(+10분)을 재계산해 배열에 저장한다.
- 근무자의 수(schedules의 길이)만큼 배열을 돌며 성공여부 함수를 호출한다
    - 만약 오늘이 8(일요일을 거쳐 월요일이 됨)이라면, 오늘을 1로 다시 정한다.
    - 만약 오늘이 토요일이나 일요일인 경우 넘어간다.
    - 출근 예정 시간보다
        - 지각했다면 0을 반환하고 다음 근무자에 대해 계산한다.
    - 출근을 올클했다면 1을 반환한다.

## 🤯 실수
### 시(時)가 바뀔 때의 로직 오류

- `958` 같은 케이스의 경우, 10분을 더하면 `1008` 으로 올림 계산이 발생한다.
- 이 때, 분이 딱 정각으로 바뀌는 경우만 생각해 다음과 같이 잘못된 로직을 구성했다.

    ```Java
    if (time % 100 == 60) {
        time = time / 100 * 100 + 100;
    }
    ```

- 분이 60보다 큰 경우, 진짜 분 단위는 살리고 시간 단위의 올림 계산이 가능하도록 다음과 같이 수정했다.

    ```Java
    if (time % 100 >= 60) {
        time += 40;
    }
    ```
